我沒有更好地與我的生命 ；如何可以貢獻 Peer.fm？

如所述的主頁上，一切都是在打開在 GitHub ； 開發任何額外的手將是輝煌的。我或發送電子郵件的東西之前發送任何拉的請求，但請不要猶豫，在 問題系統 日誌的任何 bug 報告或功能請求。

如果我決定作出貢獻，關於 Peer.fm 什麼技術細節我應該知道的嗎？

1.為了管理所有的資料，我使用的 重火力點附近 神奇即時"無伺服器"資料存儲區 （YC S11)。因此，除了一套重火力點附近驗證規則，這是純用戶端工程 ；只有甲級 JS 駭客。

2.除了重火力點附近 SDK 和 YouTube API，主要的圖書館和框架我使用的是： 谷歌關閉、 AngularJS、 糖 和 引導 (他們是所有輝煌)。

3.如果你習慣用 jQuery 的 DOM 遍歷，我基於閉包的 ClosureQuery 可能看起來有點熟悉。

什麼你可以專門使用的説明嗎？

為之作序，我打開任何擬議專案作出貢獻。說:

1.自動化測試： Peer.fm 目前有零測試覆蓋率和我有沒有計劃在不久的將來 ； 任何自己寫這可能是別人的專案來處理。

2.圖稿： 我目前借款 Peer.fm favicon （沒有特殊意義 ； Akonadi 標誌KDE 的只是令人敬畏)。

3.使用者文檔： 有無時刻。
